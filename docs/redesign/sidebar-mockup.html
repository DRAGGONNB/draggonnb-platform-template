🎯 Smart-landing dashboard

Dashboard surfaces the ONE most-likely action at the top — autopilot it (1 click) or walk through it (hands-on). Metrics still available, just below the fold. Same pattern as Phase 11 CRM Easy View, applied platform-wide.

🧭 Dynamic sidebar from tenant_modules

Activated verticals first; Content Studio + CRM secondary. Modules a tenant doesn't have don't appear at all. Drives the hardcoded array out of Sidebar.tsx; reads tenant_modules live.

🤖 Mode toggle = Autopilot / Hands-on

Per-feature, lives inside each tab. Same Easy / Advanced pattern Phase 11 introduced for CRM. Default Autopilot for first-time users; sticky preference per user.

🏷 Naming: keep "CRM"

Industry-standard, base users already know it. "Customers" is more friendly but adds a translation step.

⚙ How this gets built (Phase 12 plan preview)

  1. Drop the hardcoded navigation array in components/dashboard/Sidebar.tsx
  2. Server component: read tenant_modules + module_registry for the active org
  3. Order: Dashboard → activated verticals (by activation date) → Content Studio → CRM → Insights → Settings → [Admin if platform_admin]
  4. Each top-level section becomes one route with shadcn/ui Tabs handling sub-features
  5. Dashboard page rebuilt as smart-landing: top "Quick Action" card derived from tenant state (similar to crm_action_suggestions but cross-module)
  6. Mode toggle component extracted from Phase 11 Easy/Advanced; relabeled "Autopilot" / "Hands-on" everywhere it's used

**Diagnostic 3: PayFast token-overwrite state.**

Create `scripts/diagnostics/phase-09/check-payfast-token-state.sql`:

```sql
-- 1. Distribution of payfast_subscription_token values
SELECT
  COUNT(*) AS total_orgs,
  SUM(CASE WHEN payfast_subscription_token IS NULL THEN 1 ELSE 0 END) AS null_token,
  SUM(CASE WHEN payfast_subscription_token IS NOT NULL THEN 1 ELSE 0 END) AS has_token
FROM organizations;

-- 2. Sample tokens (safely — show first 8 chars only) to inspect format
SELECT
  id,
  name,
  LEFT(payfast_subscription_token, 8) AS token_preview,
  LENGTH(payfast_subscription_token) AS token_length,
  created_at,
  subscription_status
FROM organizations
WHERE payfast_subscription_token IS NOT NULL
ORDER BY created_at DESC
LIMIT 10;

-- 3. PayFast subscription tokens are typically UUID-shaped (36 chars with dashes)
-- pf_payment_id values are typically numeric short strings (~12 chars)
-- A length analysis tells us which we're storing
SELECT
  AVG(LENGTH(payfast_subscription_token)) AS avg_length,
  MIN(LENGTH(payfast_subscription_token)) AS min_length,
  MAX(LENGTH(payfast_subscription_token)) AS max_length,
  SUM(CASE WHEN payfast_subscription_token ~ '^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$' THEN 1 ELSE 0 END) AS uuid_shaped,
  SUM(CASE WHEN payfast_subscription_token ~ '^[0-9]+$' THEN 1 ELSE 0 END) AS numeric_only
FROM organizations
WHERE payfast_subscription_token IS NOT NULL;
```

Run via `mcp__supabase__execute_sql`. Document findings:
- If most tokens are UUID-shaped: webhook is correctly writing the subscription token. Research finding wrong.
- If most are short-numeric: token-overwrite bug confirmed; tokens are useless for PayFast amendment. Phase 10 needs to re-fetch tokens via PayFast API or wait for next ITN cycle.
- Fix recommendation: webhook should write `subscription_token` field from ITN payload (correct PayFast field name to verify in 09-04 spike), not `pf_payment_id`. Add to Phase 10 backlog.
</task>

**Diagnostic 4: Existing-org classification (test/dormant/paying).**

Create `scripts/diagnostics/phase-09/list-existing-orgs.sql`:

```sql
SELECT
  o.id,
  o.name,
  o.subdomain,
  o.subscription_tier,
  o.subscription_status,
  o.created_at,
  -- Activity signals
  (SELECT MAX(created_at) FROM agent_sessions WHERE organization_id = o.id) AS last_agent_call,
  (SELECT MAX(created_at) FROM usage_events WHERE organization_id = o.id) AS last_usage_event,
  (SELECT COUNT(*) FROM organization_users WHERE organization_id = o.id) AS user_count,
  -- Billing state
  o.payfast_subscription_token IS NOT NULL AS has_payfast_token,
  o.subscription_status IN ('active', 'trial') AS billing_active,
  -- Heuristic classification
  CASE
    WHEN o.name ILIKE '%test%' OR o.name ILIKE '%demo%' OR o.subdomain ILIKE '%test%' THEN 'test'
    WHEN o.subscription_status NOT IN ('active', 'trial') THEN 'dormant'
    WHEN (SELECT MAX(created_at) FROM agent_sessions WHERE organization_id = o.id) < NOW() - INTERVAL '30 days'
      AND (SELECT MAX(created_at) FROM usage_events WHERE organization_id = o.id) < NOW() - INTERVAL '30 days' THEN 'dormant'
    ELSE 'paying'
  END AS classification
FROM organizations o
ORDER BY o.created_at;
```

Run it. Document each org in 09-DIAGNOSTICS.md "Org Inventory" section as a table:

| Org ID | Name | Subdomain | Tier | Status | Created | Last Activity | Classification | Phase 10 Action |
|---|---|---|---|---|---|---|---|---|

Recommended Phase 10 actions per classification:
- **test:** delete (cascade via DELETE FROM organizations WHERE id = …)
- **dormant:** soft-archive (set archived_at; keep data for compliance/POPI)
- **paying:** safety-net flag; audit + migrate billing in Phase 10. Per Chris: should be zero. Verify.

Save raw classification output to `09-DIAGNOSTICS-DATA.json` for Phase 10 cleanup script to consume.

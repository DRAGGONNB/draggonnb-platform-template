# Phase 08.3: Onboarding Wizard UI

## Goal
Build the client onboarding wizard as a multi-step Next.js page with POPIA/DPA compliance, module selection, and dual Meta connection paths (Embedded Signup vs Partner WABA sharing).

### Wizard Flow
1. **Welcome** -- Path A (Start Fresh / Embedded Signup) or Path B (Link Existing WABA)
2. **Business Details** -- SA-specific: registered name, trading name, CIPC reg, +27 phone, email, address
3. **Module Selection** -- Single-select from 6 modules (CRM, Email, Social, Content Studio, Accommodation, AI Agents)
4. **POPIA & DPA** -- Scrollable agreement (Data Controller, Processor, WhatsApp Data, POPIA Rights, Breach Notification, Third Parties), checkbox acceptance gate
5. **Connect WhatsApp** -- Path A: MetaEmbeddedSignup component (OAuth popup) / Path B: WABAShareGuide component (WABA ID + share instructions)
6. **Verify & Activate** -- Summary table, activation animation, success state with next actions

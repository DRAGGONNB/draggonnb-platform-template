# Phase 08.4: Multi-Tenant Social Publishing + Token Refresh

## Goal
Make Facebook/Instagram publishing use per-tenant OAuth tokens and implement automatic token refresh to prevent 60-day token expiry interruptions.

## Prerequisites
- [ ] Phase 08.1 complete (Meta App configured, OAuth flow working)
- [ ] At least one org has Facebook Page token stored in `social_accounts`

## Tasks

### Task 1: Refactor Facebook Publishing
**File:** `lib/social/facebook.ts`
**Action:** Modify
**Details:**
- Add `orgId?: string` parameter to:
  - `publishToFacebook(orgId?, pageId, content)`
  - `publishToInstagram(orgId?, igAccountId, content)`
- Token resolution chain:
  1. If orgId provided, look up `social_accounts` for org's Facebook Page token
  2. If no org-specific token, fall back to global env var
  3. If token expired, attempt refresh before publishing
- Token refresh logic: `GET /oauth/access_token?grant_type=fb_exchange_token&fb_exchange_token={short_token}`
- Update `social_accounts.access_token` and `social_accounts.token_expires_at` on refresh

### Task 2: Token Refresh Cron Route
**File:** `app/api/meta/token-refresh/route.ts`
**Action:** Create
**Details:**
- GET handler (called by N8N cron or Vercel cron)
- Auth: API key or service-level auth (not user auth)
- Query `social_accounts` WHERE `token_expires_at < NOW() + INTERVAL '7 days'`
- For each expiring token:
  - Call Meta Graph API to refresh long-lived token
  - Update `social_accounts.access_token` and `token_expires_at`
  - Log refresh result
- Also check `tenant_modules.config` for WABA tokens nearing expiry
- On refresh failure: create admin notification (email via Resend or Telegram)
- Response: summary of refreshed/failed tokens

### Task 3: N8N Token Refresh Workflow
**File:** `n8n/wf-meta-token-refresh.json`
**Action:** Create
**Details:**
- Cron trigger: daily at 02:00 SAST
- HTTP Request to `$env.APP_URL/api/meta/token-refresh`
- On failure: send Telegram notification to admin

### Task 4: Update Facebook Publish API Route
**File:** `app/api/social/publish/facebook/route.ts`
**Action:** Modify
**Details:**
- Extract orgId from `getUserOrg()` (already available)
- Pass orgId to `publishToFacebook()`
- Error handling: if token expired and refresh fails, return specific error code

### Task 5: Update Social Account Connection
**File:** `app/api/social/accounts/route.ts`
**Action:** Modify
**Details:**
- POST handler (connect account): use Meta OAuth with page scopes
- Exchange code for long-lived Page Access Token
- Store with `token_expires_at` calculated (60 days from now)
- Instagram: linked via Facebook Page (IG Business Account)

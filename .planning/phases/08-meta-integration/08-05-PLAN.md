# Phase 08.5: Provisioning Pipeline Update

## Goal
Update the provisioning pipeline to support Meta integration: fork at step 5 based on onboarding model (A/B), configure tenant WABA, register Meta webhooks, and use tenant-specific credentials for onboarding messages.

## Prerequisites
- [ ] Phase 08.1 complete (Meta OAuth + credential storage working)
- [ ] Phase 08.2 complete (multi-tenant WhatsApp client)

## Tasks

### Task 1: Create Meta Setup Provisioning Step
**File:** `scripts/provisioning/steps/05b-meta-setup.ts`
**Action:** Create
**Details:**
- New provisioning step: `setupMetaIntegration(config)`
- Input: org_id, onboarding_model ('A' | 'B'), waba_id, phone_number_id, access_token
- Model A flow:
  1. Verify WABA ownership (GET `/{waba_id}` with access token)
  2. Register app webhook: `POST /{waba_id}/subscribed_apps` with fields: messages
  3. Set webhook URL to `https://draggonnb-platform.vercel.app/api/webhooks/whatsapp`
  4. Verify phone number is registered
  5. Store config in `tenant_modules.config`
- Model B flow:
  1. Verify partner access (GET `/{waba_id}` with DraggonnB's System User token)
  2. Confirm permissions: `whatsapp_business_messaging`, `whatsapp_business_management`
  3. Register webhook subscription (if permissions allow)
  4. Store config in `tenant_modules.config`
- Rollback: Remove webhook subscription, clear `tenant_modules.config`

### Task 2: Update N8N Provisioning Step
**File:** `scripts/provisioning/steps/05-n8n.ts`
**Action:** Modify
**Details:**
- After creating N8N workflows for tenant, inject tenant-specific vars:
  - `ORG_WABA_ID`, `ORG_PHONE_NUMBER_ID` into workflow expressions
  - Or: workflows call API with org_id header, API resolves credentials (preferred)
- Ensure WhatsApp-related workflows are only created if tenant has WABA configured
- Skip WhatsApp workflows if `onboarding_model` is null (no Meta integration)

### Task 3: Update Onboarding Step
**File:** `scripts/provisioning/steps/07-onboarding.ts`
**Action:** Modify
**Details:**
- Use `getTenantWhatsAppConfig(orgId)` for WhatsApp welcome message
- If tenant has WABA: send WhatsApp welcome via tenant's phone number
- If no WABA: skip WhatsApp welcome, send email-only welcome via Resend
- Log which channels were used for onboarding messages

### Task 4: Update Provisioning Orchestrator
**File:** `scripts/provisioning/provision-client.ts` (or equivalent)
**Action:** Modify
**Details:**
- Insert step 5b (Meta setup) after step 5 (N8N)
- Conditional execution: only run 5b if onboarding data includes Meta credentials
- Pass onboarding wizard data (model, WABA ID, tokens) through to step 5b
- Saga pattern: 5b has rollback if subsequent steps fail

# Phase 08.1: Meta App Registration & Embedded Signup Backend

## Goal
Build the backend infrastructure for Meta App integration: OAuth flows, credential storage, webhook handling for both onboarding models (A: Embedded Signup, B: Partner WABA sharing).

## Prerequisites
- [ ] Chris registers Meta App at developers.facebook.com (Business type)
- [ ] Chris provides: META_APP_ID, META_APP_SECRET, META_BUSINESS_PORTFOLIO_ID
- [ ] Chris submits for App Review with WhatsApp permissions

## Tasks

### Task 1: Meta Configuration Library
**File:** `lib/meta/config.ts`
**Action:** Create
**Details:**
- Export Meta App configuration constants (from env vars)
- Graph API version constant (v19.0 or latest)
- Scopes for Embedded Signup: `whatsapp_business_messaging`, `whatsapp_business_management`
- Scopes for Social: `pages_manage_posts`, `instagram_content_publish`
- OAuth URL builder function
- Token exchange helper (code -> access token -> long-lived token)
- Type definitions for Meta API responses (WABA info, token response)

### Task 2: Embedded Signup Flow (Model A)
**File:** `lib/meta/embedded-signup.ts`
**Action:** Create
**Details:**
- `initiateEmbeddedSignup(orgId: string)` -- generates OAuth state, returns Meta URL
- State token: JWT with orgId + nonce, stored in short-lived cookie
- OAuth URL params: client_id, redirect_uri, state, scope, response_type=code
- Redirect URI: `${APP_URL}/api/meta/callback`

### Task 3: Embedded Signup API Route
**File:** `app/api/meta/embedded-signup/route.ts`
**Action:** Create
**Details:**
- GET handler
- Auth: `getUserOrg()` required
- Generates state token, sets cookie
- Returns redirect URL to Meta OAuth
- Validation: org must exist, org must not already have WABA configured

### Task 4: OAuth Callback Route
**File:** `app/api/meta/callback/route.ts`
**Action:** Create
**Details:**
- GET handler (Meta redirects here with `code` + `state` params)
- Validate state token (JWT verify, check cookie match)
- Exchange code for short-lived token: `POST https://graph.facebook.com/v19.0/oauth/access_token`
- Exchange short-lived for long-lived token (60-day): `GET /oauth/access_token?grant_type=fb_exchange_token`
- Extract WABA details from token debug info
- Store in `tenant_modules.config` JSONB:
  ```json
  {
    "waba_id": "...",
    "phone_number_id": "...",
    "access_token": "encrypted_...",
    "token_expires_at": "2026-05-15T...",
    "business_portfolio_id": "...",
    "onboarding_model": "A",
    "setup_completed_at": "2026-03-15T..."
  }
  ```
- Redirect to `/onboarding/meta?step=verify&status=success`

### Task 5: WABA Shared Webhook (Model B)
**File:** `app/api/meta/waba-shared/route.ts`
**Action:** Create
**Details:**
- POST handler
- Webhook verification (GET with hub.challenge)
- Receives notification when client shares WABA with DraggonnB
- Payload includes: shared_waba_id, business_id, permissions granted
- Match to pending onboarding record (by WABA ID from wizard Step 5B)
- Store partner access credentials in `tenant_modules.config`
- Config shape same as Task 4 but with `onboarding_model: "B"`

### Task 6: Add Env Vars
**Action:** Configure (Vercel + VPS)
**Details:**
- Add to Vercel: `META_APP_ID`, `META_APP_SECRET`, `META_BUSINESS_PORTFOLIO_ID`
- Add to VPS `/root/.env` + docker-compose.yml: same 3 vars
- Rebuild N8N container after env var update
